## FAQ

## Frequently asked questions

Everything you need to know about Scoold. Can't find the answer you're looking for? Reach out to our support team.

### What is Scoold and how is it different from Scoold Pro?
Scoold is open-source and licensed under Apache 2.0. It's completely free. Scoold Pro is a commercial product that adds enterprise features like file uploads, SAML, SCIM provisioning, Teams/Mattermost/Slack integrations, and more.

### What does a Scoold Pro license allow me to do?
Buying a Scoold Pro license allows you to deploy one instance of Scoold Pro on one host machine (VM, container, K8s pod, etc.) and receive new releases and fixes for a period of 1 year. The license has 3 tiers - Pro basic, Pro with source, Pro unlimited. With the basic license you get the Scoold Pro binary files. The "Pro with source" license gives you the binaries and the complete source code for Scoold Pro (in a private Git repo). For larger organizations and resellers, there's the unlimited license which gives you the source code, binaries and permission to run Scoold Pro on an unlimited number of production hosts. This is ideal for resellers or organisations with larger deployments.

### Is the source code included with Scoold Pro?
The source code is only available with the "Scoold Pro with source" and "Scoold Pro unlimited" license tiers. Get in touch with us for more details.

### Can I use Scoold on-premises or in my private cloud?
Absolutely. Scoold is designed for self-hosted deployment. You can run it as a Docker container, a standalone JAR, or deploy it to AWS, Azure, GCP, or any other cloud provider. Your data never leaves your infrastructure.

### What identity providers are supported for SSO?
Scoold Pro supports SAML 2.0 SSO with any compliant identity provider, including Okta, Azure Active Directory, Auth0, OneLogin, Ping Identity, and Google Workspace. LDAP and Active Directory integration is also available for on-premises environments.
